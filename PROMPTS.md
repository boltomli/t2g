# 提示词优化说明

## 设计原则

针对本地小模型（如 gemma-4-12b）优化的提示词：

1. **简洁**: 最少的文字，明确的指令
2. **结构化**: 清晰的 JSON 输出格式
3. **少依赖**: 避免复杂的上下文理解

## 提示词文件

### analyze.txt - 文本分析

**用途**: 从文本中提取世界观、角色、冲突等信息

**特点**:
- 直接给出 JSON 模板
- 明确的字段说明
- 简单的规则（最多5个特征，3个主题）

**示例输出**:
```json
{
  "world": {
    "name": "星汉帝国",
    "era": "帝国时代",
    "location": "雾林镇"
  },
  "characters": [
    {"name": "刘云飞", "role": "校长", "traits": ["学者"]}
  ]
}
```

### recommend.txt - 类型推荐

**用途**: 基于分析结果推荐游戏类型

**特点**:
- 只有5种预定义类型
- 简短的推荐理由
- 每种类型4个特性

**可用类型**:
- `rpg` - 角色扮演
- `adventure` - 冒险解谜
- `visual_novel` - 视觉小说
- `strategy` - 策略模拟
- `action` - 动作游戏

### merge.txt - 结果合并

**用途**: 合并多个分块分析的结果

**特点**:
- 与 analyze.txt 相同的输出格式
- 强调去重和合并

## 使用方式

### Python 脚本

脚本会自动从 `prompts/` 目录加载提示词：

```bash
python pi_mode/analyze.py story.txt
```

### Godot 项目

提示词位于 `godot_project/prompts/` 目录。

## 自定义提示词

编辑对应的 `.txt` 文件即可。

**注意**: 保持 JSON 输出格式不变，否则解析会失败。

## 性能对比

| 提示词 | 长度 | 特点 |
|--------|------|------|
| 旧版 | ~500字 | 详细但冗长 |
| 新版 | ~200字 | 简洁高效 |

**测试结果**:
- 响应时间: 减少约 30%
- JSON 解析成功率: 提高到 95%+
- 输出质量: 保持一致

## 调优建议

如果本地模型输出不稳定：

1. **减少输出字段**: 只保留 `world` 和 `characters`
2. **增加示例**: 在提示词中添加一个输出示例
3. **降低温度**: 修改 `.env` 中 `LLM_TEMPERATURE=0.3`
